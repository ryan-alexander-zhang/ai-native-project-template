# Designs

This directory stores reusable design docs.
Use `TEMPLATE.md` for front matter.

## Must Include

- reusable design content that is worth linking from one or more `plan` docs

Add more when useful.

## Exclude

- task breakdown
- execution steps
- one-off implementation notes that belong inside a single `plan`

## Note

Use this directory only when a design should live outside a single `plan`.
Prefer Mermaid for design diagrams, such as class, ER, sequence, and state diagrams.
