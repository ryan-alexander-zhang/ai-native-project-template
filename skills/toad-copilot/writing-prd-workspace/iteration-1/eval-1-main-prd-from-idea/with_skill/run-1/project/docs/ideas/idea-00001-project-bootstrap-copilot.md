# Idea Brief

## One-line Summary

A project bootstrap copilot that helps teams create a consistent repo layout, core docs, and reusable agent context.

## Problem

Teams repeatedly recreate the same project scaffolding and context files by hand, which leads to drift and weak defaults.

## Target User

Developers and technical leads setting up new AI-assisted coding projects.

## Current Alternatives

Manual shell scripts, copied starter repos, and ad hoc markdown files.

## Proposed Solution

Provide a guided bootstrap flow that creates the repo layout, starter docs, and reusable project context with consistent defaults.

## Why Better

It reduces setup time and makes project structure predictable for both humans and agents.

## Why Now

AI coding workflows now depend heavily on stable repo context and repeatable setup.

## Risks

- The product could expand too far beyond the initial setup workflow.
